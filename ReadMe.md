# English for Vietnamese Tech Professionals — Learning & Recording Log

## Purpose
This repository helps Vietnamese engineers improve technical English by practicing, recording progress, and building reusable templates for real-world developer communication (PRs, standups, design discussions).

## Who this is for
- Vietnamese developers comfortable with code and logic.
- People who want practical, work-focused English: clear PR descriptions, concise standups, better code review comments, and confident spoken explanations.

## How to use
- Keep daily notes in `/daily/` using the Daily Log template.
- Add weekly summaries in `/weekly/`.
- Save short voice recordings (30–90s) in `/recordings/` to track pronunciation and fluency.
- Collect useful phrases in `phrases.md`.
- Open a PR when you add logs so teammates can give feedback.

## Learning plan (suggested)
- Daily (15–30 min): read a short article or doc, write a 3–5 sentence summary, record yourself explaining one change you made today.
- Weekly (1–2 hours): review recordings, update vocabulary, prepare a 3–5 minute tech talk about a task or bug you solved.
- Monthly: give a short demo or presentation to a colleague and collect feedback.

## Practical exercises
- Explain a recent PR in 3 sentences: Context → Change → Result.
- Write a PR description using the template: Motivation, What changed, How to test.
- Record a 60s explanation of a bug fix and compare pronunciation weekly.
- Role-play code review: write polite suggestions (examples in `phrases.md`).

## Useful phrases (examples)
- "I implemented X to fix Y. Steps to test: ..."
- "Could you review the logic in `ClassName`?"
- "My suggestion is to extract this method for clarity."
- "In standup: Yesterday I \- did X. Today I will \- do Y. Blockers: Z."

## Tips for clear technical English
- Use short sentences and active voice.
- Structure: Context → Problem → Solution → Result.
- Prefer simple words; be precise with technical terms.
- Record and re-listen: focus on clarity rather than accent.
- Ask for examples when unsure: "Could you show an example?"
