# Issue Tips — quick frameworks for responding to problems

Purpose

This folder stores bite-sized frameworks and example responses you can reuse when facing issues (PR comments, incidents, customer emails, interview answers).

Guiding principle

Start simple first. Use a minimal structure (2–3 lines) so you can respond clearly and quickly. When you have time, expand using a full framework like PEEL (Point — Evidence — Example — Link).

What you’ll find here

- `SIMPLE-FIRST.md` — a minimal, fast template to use immediately.
- `PEEL-template.md` — the full PEEL framework with tips for tech contexts.
- `examples/peel-pr.md` — a PR explanation example using PEEL.
- `examples/peel-incident.md` — incident response example (short then expanded).
- `examples/peel-customer-email.md` — customer-facing reply using PEEL.

How to use

1. Pick `SIMPLE-FIRST.md` for a fast reply (under 3 lines).
2. If you need a fuller explanation or document, open `PEEL-template.md` and the relevant example.
3. Copy, edit, and paste into your reply; keep the language plain and polite.

Contributing

- Add more example files under `examples/` for real situations you encounter.
- File names should be short and searchable, e.g. `examples/peel-code-review.md`.
