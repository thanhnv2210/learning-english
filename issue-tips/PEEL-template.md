# PEEL Framework for Technical Responses

PEEL = Point — Evidence — Example — Link

Use PEEL for longer explanations (post-mortems, RFCs, customer emails).

Template

1) Point (1 sentence): State your main point or conclusion.
2) Evidence (1–2 sentences): Provide data, logs, or what you observed.
3) Example (1–2 sentences): Show a concise example or reproduce steps.
4) Link / Next step (1 sentence): Provide a link to PR/issue or list the next action.

Tips for tech contexts

- Keep the "Evidence" short: error messages, timestamps, test names.
- The "Example" can be a minimal code snippet or a short reproduction step list.
- The "Link" should point to the canonical source (issue, PR, dashboard).
- When writing for non-technical readers, add one simple sentence explaining why the issue matters.

Minimal PEEL (if you’re pressed for time)

- Point + Evidence + Link (skips the Example).

Example skeleton

Point: <one-line conclusion>
Evidence: <short log / failure rate / test name>
Example: <minimal reproduction or code snippet>
Link / Next step: <PR link / action item / timeline>
