# Special Issues — context-specific situations

Purpose: Handle unusual or higher-stakes situations where language needs to be precise.

Examples

- `customer-calls.md` — language and scripts for customer calls, polite escalation phrases.
- `interviews.md` — how to present accomplishments, common interview questions.
- `system-design-talks.md` — phrases to describe architecture and trade-offs.

Format

- Each file should include: scenario, suggested script, variations, and practice prompts.

Example: `customer-calls.md`

- Scenario: Customer calls about a failed payment
- Script: "Hello, my name is X from engineering. I understand you're seeing a failed payment for transaction ID ..."
- Variation: Shorter/longer, more technical, non-technical.
- Prompt: "Role-play a customer call where the customer reports a failed payment. You are the engineer explaining the steps you'll take."
